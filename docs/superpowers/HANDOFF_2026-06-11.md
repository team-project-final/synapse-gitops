## 3. 디버깅 핵심 (추측 금지 → 증거 확정)

1. **bring-up이 EKS서 ES 못 띄움(#180)**: fsGroup 누락(EBS root 소유→node.lock 실패 CrashLoop; **fsGroup chown은 새 pod 마운트시만 적용**→기존 pod delete 필요)·ES/schema-registry standalone App 미배포·gp3 SC 순서·es-reindex 레이스.
2. **Git Bash 함정(#181)**: Windows kubectl.exe가 MSYS 경로(`/tmp`) KUBECONFIG 무시→localhost:8080 폴백. cygpath -m 변환. + stale tf lock·갓생성 클러스터 첫 터널 폴링 부족.
3. **engagement CrashLoop(추측 오진 교훈)**: 이미지 회복 후에도 CrashLoop → 처음 #164 패턴으로 **추측**해 cpu/startupProbe 올림(#187)→안 고쳐짐→원복(#188). `describe` 증거로 실원인 확정: `Liveness probe 401`·`Started in 71.5s`·exitCode 143 → **앱 actuator health 401(Spring Security)**가 livenessProbe SIGTERM. 앱레포 **engagement-svc#43**(permitAll `/actuator/health/**`) 수정→1.0.1 빌드→배포→liveness 200. **교훈: 증상→수정 직행 금지, describe/로그/curl/정상사례 대조 증거로만.**
4. **#191 staging 태그**: engagement `dev-latest`(semver만 push라 미생성)·learning-card `ab67c3c2`(미빌드) = ECR 부재 stale 핀 → ImagePullBackOff. dev 정합 ECR 실태그로 정정(1.0.1·e5e27ee2, digest 검증).
