## 1. 목적 · 출처

W4 진입 전 백로그를 한 곳에 통합한다. 세 출처를 합친다:

1. **원래 W4 계획** — `PRD_W4`(FR-GO-401~408) + `WORKFLOW_W4`(Step 9 prod+승인게이트, Step 10 롤백/백업)

## 4. prod 환경 — W4 착수 시 결정 필요 (OPEN)

`WORKFLOW_W4` Step 9 1.1 사전분석의 미결정 사항. **여기서 결정하지 않음** — W4 brainstorming으로 확정:

- **prod 클러스터** — 별도 클러스터(격리·비용↑) vs dev 클러스터 내 네임스페이스 논리분리(비용↓·격리 약함)
- **승인 게이트** — ArgoCD Manual Sync vs GitHub Environment Approval
- **권한 분리** — gitops-admin 그룹 IAM/RBAC 모델
- **RTO/RPO 목표** — 예: RTO 30분 / RPO 1시간 (Step 10, team-lead 합의)
- **prod 도메인/ACM** — A4와 동일하게 실 Route53 zone 필요

## 5. 조건부 EKS 라이브 사이클 (비용 batching)

W3에서 이월된 라이브 검증(A3/A4/A5 + platform 5/5)과 W4 prod 배포는 **같은 라이브 사이클에 묶어 과금 1회로** 처리한다. `scripts/bring-up.sh`(W3 검증된 11 phase) 재사용.

- 라이브 진입 전: D-039(ESO role import/삭제) 선처리.
- A4는 실 도메인/Route53 zone이 없으면 그대로 다음으로 이월.
- 종료 시 반드시 `terraform destroy` — S3 state bucket + DynamoDB lock만 유지.

**비용**: ~$0.41/hr (EKS+RDS+MSK+Redis+OpenSearch).
