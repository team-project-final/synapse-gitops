4.1 State Backend 수동 생성

chicken-and-egg: backend(S3+DynamoDB)가 있어야 terraform init이 된다 → 수동 선생성.

S3 State Bucket

aws s3api create-bucket `
  --bucket synapse-terraform-state `
  --region ap-northeast-2 `
  --create-bucket-configuration `
    LocationConstraint=ap-northeast-2
# + versioning · AES256 암호화
# + public access 4종 차단

DynamoDB Lock Table

aws dynamodb create-table `
  --table-name synapse-terraform-locks `
  --attribute-definitions `
    AttributeName=LockID,AttributeType=S `
  --key-schema `
    AttributeName=LockID,KeyType=HASH `
  --billing-mode PAY_PER_REQUEST
⚠️ 자원 생성 시각을 기록하고, 학습 완료 후 즉시 destroy.
S3 이름은 글로벌 unique. 충돌 시 main.tf PR. DynamoDB는 wait 까지.

20~25분

plan 마지막 줄에서 to destroy: 0 확인 필수 — 아니면 무엇이 삭제되는지 보고 결정.

apply 진행 순서

1. VPC / Subnet / IGW / Route Table        (1~2분)
2. NAT Gateway ×2 (EIP) · IAM · SG          (2~10분)
3. EKS Cluster  ← 가장 오래               (5~15분)
4. RDS · ElastiCache (병렬) · MSK · OpenSearch
5. EKS Node Group → Helm(ArgoCD)            (마지막)
네트워크(vpc)가 먼저, 그 위에 컴퓨트(eks), 데이터 계층 동일 사설망, addons/IRSA는 클러스터 후.

4.4 자주 막히는 곳

증상원인 → 해결
Node Group launch 실패 (Free Tier)결제수단 verification 미완 → Account 정보 채우고 24~72h 대기, 또는 kind 로컬
MSK SubscriptionRequired신규 계정 MSK 미활성 → 콘솔에서 "Amazon MSK" 페이지 진입 후 재apply
OpenSearch service-linked role 없음aws iam create-service-linked-role --aws-service-name opensearchservice.amazonaws.com
Error acquiring the state lock비정상 종료 잔여 lock → terraform force-unlock <ID>
EIP AddressLimitExceededService Quotas 증설, 또는 vpc.tf NAT 1개(single-AZ)
전체 케이스 → runbooks/step3-terraform-apply.md · troubleshooting-infra.md
대부분 신규 계정 1회성. 일시적 throttle(Redis 등)은 재apply.
